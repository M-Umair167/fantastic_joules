## What This Project Does

This repository contains our reproduction of the data analysis sections of the IMC '25 paper above. The paper investigates router energy demand using a unique dataset from Switch, a Tier-2 ISP in Switzerland, and provides the first publicly validated router power models.

We reproduced:
- **Figure 1** — Total network power and traffic time series
- **Figure 2a/2b** — Power efficiency trends (Broadcom ASIC vs. router datasheets)
- **Figure 5** — PSU efficiency curves and 80 Plus standards
- **Figure 6** — PSU efficiency scatter across all router models
- **Table 1** — Datasheet vs. measured power comparison (all 8 routers, exact match)
- **Table 3** — PSU efficiency savings estimates (exact match)
- **Table 4** — PSU right-sizing savings estimates (exact match)
- **Section 7 results** — Traffic energy cost calculations

## Setup Instructions

### Prerequisites

- Python **3.9 or higher**
- pip
- Git

### Step 1 — Clone or Download the Project

If you have Git:
```bash
git clone <your-repo-url>
cd Fantastic-Joules
```

Or simply download and extract the ZIP, then open a terminal inside the `Fantastic-Joules/` folder.

---

### Step 2 — Create a Virtual Environment

**Windows (PowerShell):**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

**Mac / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

You should see `(venv)` appear at the start of your terminal prompt.

---

### Step 3 — Install Dependencies

```bash
pip install -r requirements.txt
```

This installs all required packages including:

| Package | Version | Purpose |
|---|---|---|
| pandas | latest | Data loading and manipulation |
| numpy | latest | Numerical operations |
| scipy | latest | PSU load interpolation (v2 notebook) |
| plotly | 5.21.0 | All visualizations |
| kaleido | 0.2.1 | Saving figures as PDF/PNG |
| notebook | ≥5.3 | Jupyter notebook server |
| ipykernel | latest | Jupyter kernel |
| ipywidgets | ≥7.5 | Interactive widgets |
| nbconvert | latest | Notebook conversion |
| jupyter | latest | Full Jupyter suite |
| pyyaml | latest | YAML parsing |

> **Note:** `kaleido` is pinned to **0.2.1** intentionally. Newer versions (1.x) are incompatible with `plotly==5.21.0` and will cause `fig.write_image()` to fail. Do not upgrade kaleido.

---

### Step 4 — Select Python Interpreter in VS Code (if using VS Code)

Press `Ctrl+Shift+P` → type **Python: Select Interpreter** → choose:
```
Python 3.x.x ('venv': venv)  —  ./venv/Scripts/python.exe
```

This ensures VS Code uses the venv and not the system Python.

---

## Running the Notebooks

### Option A — Run All at Once (Recommended for quick check)

With the venv activated:

```bash
cd Fantastic-Joules
python run_all.py
```

This executes both `datasheet-analysis_IMC25.ipynb` and `PSU-efficiency-analysis_IMC25.ipynb` sequentially and prints all output to the terminal. Generated PDF figures are saved to `output/2025_IMC/figures/`.

---

### Option B — Interactive Jupyter Notebook

```bash
jupyter notebook
```

A browser tab will open. Run the notebooks in this order:

1. `datasheet-analysis_IMC25.ipynb` → generates Figure 1, Figure 2, Table 1 output
2. `PSU-efficiency-analysis_IMC25.ipynb` → generates Figure 5, Figure 6, Table 3, Table 4
3. `PSU-efficiency-analysis_v2.ipynb` → revised version with scipy-based interpolation

> **Important:** After running, restart the kernel between notebooks if you encounter any variable conflicts.

---

### Option C — Run in VS Code

Open any `.ipynb` file in VS Code. Make sure the kernel shown in the top-right is:
```
Python (venv)  —  ./venv/Scripts/python.exe
```

Click **Run All** or run cells individually with `Shift+Enter`.

## Known Issues & Fixes Applied

### Issue 1 — Kaleido / Plotly Version Conflict
**Symptom:**
```
ValueError: Image export using the "kaleido" engine requires the kaleido package...
UserWarning: You have Plotly version 5.21.0, which is not compatible with this version of Kaleido (1.3.0)
```
**Fix:** Downgraded kaleido to `0.2.1`. Already applied in `requirements.txt` — just run `pip install -r requirements.txt` and it installs the correct version automatically.

---

### Issue 2 — Missing numpy / scipy
**Symptom:** `ModuleNotFoundError: No module named 'numpy'` or `No module named 'scipy'`  
**Fix:** Both packages are now included in `requirements.txt`. Already resolved.

---

### Issue 3 — Kernel Uses Old Package Versions After Reinstall
**Symptom:** Error persists even after fixing packages.  
**Fix:** Restart the Jupyter kernel. In VS Code: `Ctrl+Shift+P` → **Notebook: Restart Kernel**. In Jupyter browser: **Kernel → Restart**.

---

### Issue 4 — PowerShell Execution Policy (Windows)
**Symptom:**
```
.\venv\Scripts\Activate.ps1 cannot be loaded because running scripts is disabled
```
**Fix:** Run this once in PowerShell as Administrator:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## What Could Not Be Reproduced

| Section | Reason |
|---|---|
| Figure 4 (Autopower vs PSU vs Model traces) | Raw Autopower hardware data is private — Switch ISP could not publicly release it |
| Table 2 (Power models) | Requires physical lab experiments on production routers (NetPowerBench) |
| Figure 9 (Zoomed model predictions) | Depends on Table 2 power models |
| Section 8 (Link sleeping / Hypnos) | Requires private Switch network topology and router inventory |

These gaps are explicitly documented by the original authors in §9.2 and §11 (Ethics) of the paper. Everything in the **public artifact** was successfully reproduced.
